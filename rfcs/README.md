# RFCs

RFCs define proposed user-visible contracts and on-disk behavior. They are not
accepted merely because implementation has started.

## Statuses

- **Proposed**: under active design and subject to incompatible change.
- **Accepted**: approved for implementation; material decisions become ADRs.
- **Rejected**: considered and declined.
- **Superseded**: replaced by a later RFC.
